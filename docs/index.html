Local Claude Code remote control

Claude Bridge

A small, mobile-friendly web app for controlling your local Claude Code sessions from a browser, scheduling prompts, attaching files, and watching run events.

View on GitHub Quick start Tailscale setup Testing Security model

Why It Exists

Use Claude from mobile

Open a private Tailscale URL and send prompts to your local Claude Code sessions from your phone.

Keep sessions organized

Sessions are grouped by project folders, similar to Codex, so work stays easy to scan.

Schedule work

Prepare a prompt and have it sent later, even after restarting the Bridge.

Attach files

Upload files, preview images, paste screenshots, and pass attachments to Claude CLI.

Choose permissions

Select model, permission mode, and allowed tools before sending a run.

Watch progress

Stream run events into a small timeline and stop running jobs when needed.

Quick Start

Run Claude Bridge on the same machine where Claude Code is installed and authenticated.

Windows

cd "<path-to-ClaudeBridge>"
.\ClaudeBridge.cmd

macOS

cd "<path-to-ClaudeBridge>"
bash ./ClaudeBridge.sh

When the app starts, the PowerShell or terminal window prints the login URL and token you need to sign in.

Then open:

http://127.0.0.1:8765/

Mobile Access

Use Tailscale Serve for private mobile access. The launcher configures Serve when the Tailscale CLI is installed and connected.

# Windows
.\ClaudeBridge.cmd

# macOS
bash ./ClaudeBridge.sh

Open your Tailscale machine URL on your phone:

http://YOUR-MACHINE.YOUR-TAILNET.ts.net/

Tailscale Setup

1. Install and sign in

Install Tailscale on the computer and phone, then sign in to the same tailnet on both devices.

2. Use MagicDNS

Find the computer's MagicDNS name in Tailscale, then open it from the phone as an HTTP URL.

3. Run the launcher

Windows uses ClaudeBridge.cmd. macOS uses bash ./ClaudeBridge.sh.

Windows

winget install --id Tailscale.Tailscale -e
.\ClaudeBridge.cmd

If Serve needs administrator access, run once from Administrator PowerShell:

.\setup-tailscale-serve.cmd

macOS

# Install Tailscale from tailscale.com/download/mac
tailscale status
bash ./ClaudeBridge.sh

If Serve needs manual setup, run once:

bash ./setup-tailscale-serve.sh

Full details are in TAILSCALE.md. Keep Claude Bridge private to your Tailscale network; do not forward router ports to it.

How It Works

Claude Bridge does not automate the Claude Desktop window. It runs the Claude CLI locally and resumes selected Claude Code sessions.

claude -p "your message" --resume SESSION_ID

Local Data

  • config.json: local login token
  • claude_bridge.sqlite3: runs, jobs, events
  • uploads/: local uploaded attachments

Testing With Teammates

Each tester should run Claude Bridge on their own computer with their own Claude Code login. Do not share your local token, config.json, database, uploads folder, or personal Tailscale URL.

Send these

Repository link, README.md, TESTING.md, TAILSCALE.md, and SECURITY_REVIEW.md.

Ask them to verify

Login, session list, session history, model and permission defaults, send, schedule, delete schedule, attachments, and mobile Tailscale access.

Ask for reports

OS, browser, Claude Code version, local or Tailscale URL, screenshot, and terminal output. Never ask for tokens.

Security Model

Claude Bridge is currently best used as a private Tailscale-only MVP. Do not expose it directly to the public internet with only a shared token.

Good for private use

Tailscale, trusted users, local machine, and narrow tool permissions.

Not public-ready yet

Needs per-user auth, rate limiting, audit logs, token rotation, and stronger approval handling.

Be careful with tools

Broad Bash/Edit/Write permissions can affect files on the host machine.

Claude Bridge is an independent local tool for Claude Code users.
